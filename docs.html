Getting Started

🚀 Quick Start: Add @agents before any prompt in your AI tool and get expert-level, role-specific responses instantly!

The AI Agent Orchestrator Framework transforms generic AI responses into expert-level, persona-driven outputs. Instead of getting generic code suggestions, you get specialized guidance from virtual experts like Senior Engineers, Security Engineers, DevOps Engineers, and more.

Installation

Method 1: Direct Download

# Download the framework file
curl -o agents.md https://raw.githubusercontent.com/IlamSingleBrainCell/ai-agent-orchestrator/main/agents.md

# Place in your project root
mv agents.md /your-project-root/

Method 2: Git Clone

git clone https://github.com/IlamSingleBrainCell/ai-agent-orchestrator.git
cd ai-agent-orchestrator
cp agents.md /your-project-root/
✅ That's it! No configuration needed. The framework works immediately.

Basic Usage

Using the framework is incredibly simple. Just add @agents before your prompt:

Before (Generic AI)

Result: Basic, generic implementation

After (Expert Persona)

"@agents Create a secure login system with OAuth2"

Result: Production-ready implementation from Security Engineer + Backend Engineer personas

Advanced Usage

Multi-Persona Tasks

Complex tasks automatically get multiple expert personas:

@agents "Build complete e-commerce checkout flow"

→ Automatically assigns:
  • Product Owner (user stories)
  • Frontend Engineer (UI implementation)  
  • Backend Engineer (payment processing)
  • Security Engineer (payment security)
  • QA Lead (testing strategy)

Explicit Persona Selection

@agents "As Security Engineer and DevOps Engineer, review this deployment script"

Troubleshooting

⚠️ AI Tool Not Recognizing Framework?
  • Ensure agents.md is in your project root
  • Try explicit reference: "Using the agents framework..."
  • Check if your AI tool supports file context
💡 Getting Generic Responses?
  • Use @agents prefix consistently
  • Be more specific with your prompts
  • Include relevant project context
  • Try explicit persona requests

Common Issues

Issue: Wrong Persona Assigned

Solution: Use explicit persona request:

@agents "As DevOps Engineer, create deployment pipeline"

Issue: Need Multiple Personas

Solution: Request specific combination:

@agents "As Security Engineer and Backend Engineer, implement OAuth2"

Issue: Framework Not Loading

Solution: Include explicit context:

"Reference the agents.md framework and respond as appropriate persona: [your prompt]"
